# 报销信息自动化处理系统 v1.2 – 需求规格说明书

> 文档状态：已定稿 | 对应系统版本：v1.2

## 1. 背景与目标

针对基地讲师路费报销流程中存在的“发票命名乱、人工计算累、数据对账难”等痛点，搭建轻量化线上报销系统。

**核心目标**：实现“填报即规范、数据自动算、财务一键导”，确保每一笔金额与发票文件精准对齐，降低财务核对成本，提升讲师报销体验。

## 2. 功能需求

### 2.1 身份与权限逻辑

- **登录校验**：讲师通过系统号/手机号登录。登录后，表单中的「讲师姓名」「系统号」「手机号」由系统自动预填且不可修改。
- **状态锁定**：报销记录状态分为“已提交”“已审核”“已作废”。管理员导出数据后，对应记录状态自动变更为“已锁定”，讲师不可再修改或删除。

### 2.2 表单字段与附件处理

报销以 **一张发票 / 一个行程** 为基本单位，避免多对多导致的对账混乱。

- **行程信息**：发票日期（必填）、行程起点（必填）、行程终点（必填）、车辆类型（必填）。
- **费用明细**：
  - 打车费金额（必填，精确到小数点后两位，≥0）
  - 是否含路桥费（是/否）。若选“是”，则路桥费金额为必填项，且数值≥0
  - **合计金额**：打车费 + 路桥费（只读，由系统实时计算，后端二次校验）
- **附件处理（关键）**：
  - 文件格式：支持 JPG、PNG、PDF。
  - 上传数量：单条报销记录限 1–3 张附件。
  - 合规引导：上传界面需强提醒“请确保发票金额、日期和公章清晰”。
  - 完整性校验：后端需检测上传文件是否完整，拦截 0KB 或损坏的空文件。

### 2.3 数据存储与导出规则

- **行程唯一标识符（Record_ID）**：每条报销记录在提交时自动生成唯一 ID。该 ID 需同时映射在：
  1. 数据库记录行（主键）
  2. 导出 Excel 的首列
  3. 发票附件的重命名文件名中
- **自动命名规则**：`姓名_系统号_发票日期_Record_ID_序号.扩展名`
- **一键导出**：导出 Excel 需包含所有字段，且增加“发票对应文件名”列，确保财务通过 Excel 记录能直接定位到文件夹中的文件。

## 3. 字段定义表（核心字段）

| 字段名 | 类型 | 必填 | 校验逻辑/备注 |
| :--- | :--- | :--- | :--- |
| **Record_ID** | String | 是 | 系统生成唯一主键，用于记录、Excel、发票文件三者关联对账 |
| **Speaker_Name** | String | 是 | 登录信息自动预填，页面只读不可修改 |
| **Total_Amount** | Decimal(5,2) | 是 | 自动计算（Taxi_Fee + Toll_Fee），后端二次校验 |
| **Invoice_Files** | String[] | 是 | 存储文件相对路径，单记录最多 3 个，单文件＜5MB |
| **Status** | Enum | 是 | 已提交 / 已审核 / 已作废 / 已锁定 |

> 完整字段参见系统数据库设计，此处仅列出核心对账字段。

## 4. 业务规则与异常处理

| 场景 | 逻辑校验规则 / 异常处理 |
| :--- | :--- |
| **防重校验** | 同一讲师 + 同一发票日期 + 同一金额 + 相近行程起点/终点（可选），系统弹出“疑似重复提交”弹窗，需讲师确认后方可继续。 |
| **弱网环境** | 附件上传过程中网络中断，系统支持最多 3 次自动重传，失败后需保留已填写字段（存为草稿）。 |
| **超额提醒** | 单笔报销金额若超过设定阈值（如 500 元），需增加弹窗二次确认，防止误填。 |
| **修改权限锁定** | 只要财务执行过“导出”操作，对应记录的 Status 自动变更为“已锁定”，前端关闭编辑入口。 |

## 5. 系统性能及安全要求

- **并发处理**：支持 50 人同时上传发票，系统响应时间 ≤ 3 秒。
- **数据安全**：导出功能必须强制进行管理员二次鉴权（如再次输入密码或短信验证码）。
- **异步打包**：发票批量下载需在后台队列执行，完成后通过页面气泡或系统消息通知用户，避免阻塞浏览器。

## 6. 验收标准

1. **链路闭环**：通过 Excel 中的 `Record_ID` 能在发票包中找到 100% 对应的文件。
2. **金额零误差**：经 1000 次模拟填报测试，合计金额计算准确率达到 100%。
3. **权限隔离**：非本人或非管理员无法通过 URL 或 ID 越权访问附件。

## 7. 后续规划

- **v1.3**：接入 OCR 技术，自动提取发票上的代码和校验码，实现物理查重。
- **v1.4**：适配企业微信/钉钉移动端 H5 填报，支持在外讲师便捷提交。
